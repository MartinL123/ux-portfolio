Soutenance 09 · Prototypage HTML

Prototypes interactifs
en HTML natif

Concevoir des wireframes puis un prototype haute fidélité entièrement en HTML/CSS pour le service familial Les Bambinets — sans outil de maquette.

👨‍👩‍👧 Les Bambinets 3 livrables HTML
Contexte projet Les Bambinets est un service dédié aux familles avec enfants. La contrainte du module : produire des prototypes interactifs uniquement en HTML/CSS, sans Figma ni outil de maquette. L'objectif est de démontrer qu'un designer UX peut prototyper directement dans le medium final — et que ce résultat est plus fidèle et plus exploitable par les développeurs.
Les 3 livrables HTML
De la structure au pixel perfect
📐
Wireframes v2
Maquettes basse fidélité en HTML : structure des pages, hiérarchie de l'information, logique de navigation. Version 2 après intégration des premiers retours.
🎨
Prototype Hi-Fi
Prototype haute fidélité avec design finalisé : couleurs, typographies, espacements, micro-interactions CSS. Ressemble et se comporte comme le produit final.
🎤
Présentation de soutenance
Support de présentation HTML pour le jury : contexte du projet, démarche UX, choix de design argumentés, démonstration des prototypes. Un format cohérent avec les livrables.
Pourquoi prototyper en HTML ?
Le medium le plus fidèle
  • Fidélité maximale. Un prototype HTML se comporte exactement comme le produit final : scroll réel, hover states, focus clavier, transitions CSS. Aucun outil de maquette ne peut égaler ça.
  • Livrable exploitable par les devs. Au lieu d'un PDF ou d'un lien Figma, le développeur reçoit du code HTML/CSS qu'il peut inspecter, copier et adapter directement.
  • Tests utilisateurs plus représentatifs. Tester un prototype HTML sur un vrai navigateur génère des réactions authentiques — les utilisateurs ne "font semblant" pas de cliquer, ils cliquent vraiment.
  • Itération rapide. Modifier une couleur, un espacement ou un composant CSS prend quelques secondes — pas de re-export, pas de sync, pas de plugins.
